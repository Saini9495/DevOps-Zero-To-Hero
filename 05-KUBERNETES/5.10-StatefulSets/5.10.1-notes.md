# StatefulSets in Kubernetes

## What is a StatefulSet?

A StatefulSet is a Kubernetes workload resource used to manage stateful applications.

Unlike Deployments, StatefulSets provide:

* Stable Pod Names
* Stable Network Identity
* Persistent Storage
* Ordered Pod Creation
* Ordered Pod Deletion

StatefulSets are generally used for:

* MySQL
* PostgreSQL
* MongoDB
* Cassandra
* Elasticsearch
* Redis Cluster

---

# Why Not Use Deployment?

Deployments are designed for stateless applications.

Example:

Pods created by Deployment:

```text
nginx-7c98d6f7d5-abcde
nginx-7c98d6f7d5-fghij
nginx-7c98d6f7d5-klmno
```

If a Pod dies:

```text
nginx-7c98d6f7d5-xyz12
```

A new random name is assigned.

For databases this is a problem because:

* Database nodes need fixed identities.
* Data should not be lost.
* Network addresses should remain stable.

StatefulSet solves these problems.

---

# StatefulSet Features

## Stable Pod Names

Pods get predictable names:

```text
mysql-0
mysql-1
mysql-2
```

Even after restart:

```text
mysql-0
mysql-1
mysql-2
```

Names remain unchanged.

---

## Persistent Storage

Each Pod gets its own volume.

Example:

```text
mysql-0 → pvc-mysql-0
mysql-1 → pvc-mysql-1
mysql-2 → pvc-mysql-2
```

Data remains even if Pod is recreated.

---

## Ordered Deployment

Pods start sequentially.

```text
mysql-0
   ↓
mysql-1
   ↓
mysql-2
```

Kubernetes waits for mysql-0 to become Ready before creating mysql-1.

---

## Ordered Termination

Pods terminate in reverse order.

```text
mysql-2
   ↓
mysql-1
   ↓
mysql-0
```

# StatefulSet vs Deployment

| Feature          | Deployment       | StatefulSet |
| ---------------- | ---------------- | ----------- |
| Pod Name         | Random           | Fixed       |
| Storage          | Shared/Temporary | Persistent  |
| Network Identity | Dynamic          | Stable      |
| Pod Creation     | Parallel         | Ordered     |
| Pod Deletion     | Random           | Ordered     |
| Database Support | No               | Yes         |

---

# Real-Life Analogy

Imagine a classroom.

Deployment:

```text
Student A leaves.
Another student comes.

Identity doesn't matter.
```

StatefulSet:

```text
Roll No 1
Roll No 2
Roll No 3
```

Even if Roll No 1 leaves and returns, he keeps the same identity.

This is how StatefulSets work.

# Headless Service

StatefulSet requires a Headless Service.

Normal Service:

```text
ClusterIP
```

Headless Service:

```yaml
clusterIP: None
```

This allows direct communication with individual Pods.

Example:

```text
mysql-0.mysql-service.default.svc.cluster.local

mysql-1.mysql-service.default.svc.cluster.local

mysql-2.mysql-service.default.svc.cluster.local
```

Every Pod gets its own DNS record.

# Interview Questions

## What is StatefulSet?

StatefulSet is a Kubernetes workload resource used to manage stateful applications while providing stable identities and persistent storage.

---

## Why do we need StatefulSet?

Stateful applications such as databases require:

* Fixed Pod Names
* Persistent Volumes
* Stable Network Identities

StatefulSet provides these features.

---

## Difference Between StatefulSet and Deployment?

Deployment is used for stateless applications.

StatefulSet is used for stateful applications.

---

## Why is Headless Service Required?

Headless Service creates DNS entries for individual Pods.

This allows Pods to communicate directly.

---

## Can StatefulSet work without Persistent Volumes?

Yes.

But it defeats the purpose because data will be lost after Pod recreation.

---

## Give Real World Examples of StatefulSet.

* MySQL
* PostgreSQL
* MongoDB
* Cassandra
* Elasticsearch
* Redis Cluster

# Summary

* StatefulSet manages stateful applications.
* Provides stable Pod identities.
* Provides persistent storage.
* Uses Headless Services.
* Supports ordered deployment and termination.
* Ideal for databases and clustered applications.
