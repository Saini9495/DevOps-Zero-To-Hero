# Kubernetes Pods

## What is a Pod?

A Pod is the smallest deployable unit in Kubernetes.

A Pod can contain:

* One Container (Most Common)
* Multiple Containers (Sidecar Pattern)

Pod acts as a wrapper around containers.

---

## Why Pods?

Kubernetes does not directly manage containers.

Instead:

Kubernetes в†’ Pod в†’ Container

This provides:

* Scalability
* Self Healing
* Networking
* Storage Management

## Pod Lifecycle

1. Pending
2. Running
3. Succeeded
4. Failed
5. Unknown

### Pending

Pod created but container image not downloaded.

### Running

Pod successfully running.

### Succeeded

Container completed successfully.

### Failed

Container crashed or exited with error.

### Unknown

Node communication lost.

## Pod Storage

Containers are temporary.

If Pod is deleted:

Data is lost.

For persistent storage:

* Persistent Volume
* Persistent Volume Claim

are used.

## Real Life Example

Imagine a Food Delivery Rider.

Container = Food Package

Pod = Delivery Bag

Kubernetes manages the Delivery Bag (Pod), not the individual package.

---

## Advantages of Pods

* Smallest deployable unit
* Easy scaling
* Container grouping
* Shared networking
* Shared storage

---

## Disadvantages

* Ephemeral
* Direct pod usage not recommended in production
* Use Deployment instead

---

## Interview Questions

### What is a Pod?

A Pod is the smallest deployable unit in Kubernetes that wraps one or more containers.

---

### Can a Pod have multiple containers?

Yes.

Multiple containers can run inside a single Pod and share storage and networking.

---

### Does each Pod get an IP?

Yes.

Every Pod receives its own IP Address.

---

### What happens if a Pod dies?

If managed by Deployment or ReplicaSet, Kubernetes creates a new Pod automatically.

---

### Difference Between Pod and Container?

Container:

* Runtime instance

Pod:

* Wrapper around containers managed by Kubernetes
