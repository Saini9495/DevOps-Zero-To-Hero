# ReplicaSets

## What is a ReplicaSet?

ReplicaSet is a Kubernetes object that ensures a specified number of pod replicas are always running.

If a Pod crashes or gets deleted, ReplicaSet automatically creates a new Pod.

---

## Why Do We Need ReplicaSets?

Without ReplicaSet:

Pod Deleted
    ❌
Application Down

With ReplicaSet:

Pod Deleted
    ↓
ReplicaSet Detects Failure
    ↓
New Pod Created
    ✅

Application Remains Available

---

## Features

- Maintains desired number of Pods
- Self-healing
- Supports scaling
- Uses labels and selectors
- Managed by Deployments

---

## Architecture

ReplicaSet
    │
    ├── Pod-1
    ├── Pod-2
    └── Pod-3

Desired Replicas = 3

If Pod-2 crashes:

ReplicaSet
    │
    ├── Pod-1
    ├── Pod-3
    └── New Pod

---

## How ReplicaSet Works

Step 1:
User creates ReplicaSet

Step 2:
ReplicaSet creates Pods

Step 3:
ReplicaSet continuously monitors Pods

Step 4:
If a Pod fails, ReplicaSet creates a replacement Pod

## Components of ReplicaSet

### 1. apiVersion

Defines API version.

```yaml
apiVersion: apps/v1
```

### 2. kind

Defines object type.

```yaml
kind: ReplicaSet
```

### 3. metadata

Contains object information.

```yaml
metadata:
  name: nginx-rs
```

### 4. spec

Contains ReplicaSet configuration.

```yaml
spec:
```

### 5. replicas

Defines desired number of Pods.

```yaml
replicas: 3
```

### 6. selector

Identifies Pods managed by ReplicaSet.

```yaml
selector:
  matchLabels:
    app: nginx
```

### 7. template

Pod blueprint.

```yaml
template:
```

---

## Label Selector

ReplicaSet uses labels to identify Pods.

Example:

```yaml
labels:
  app: nginx
```

Selector:

```yaml
selector:
  matchLabels:
    app: nginx
```

Labels and selectors must match.

---

## Scaling ReplicaSets

Current Replicas = 3

Scale to 5:

```bash
kubectl scale rs nginx-rs --replicas=5
```

Result:

Pod-1
Pod-2
Pod-3
Pod-4
Pod-5

## Self-Healing Example

Delete Pod:

```bash
kubectl delete pod nginx-rs-abc123
```

ReplicaSet automatically creates:

```bash
nginx-rs-xyz456
```

Application remains available.

## ReplicaSet vs ReplicationController

| Feature | ReplicaSet | ReplicationController |
|----------|------------|----------------------|
| Label Selector | Yes | Limited |
| Modern Kubernetes | Yes | No |
| Recommended | Yes | No |
| Deployment Support | Yes | No |

---

## ReplicaSet vs Deployment

| ReplicaSet | Deployment |
|------------|------------|
| Manages Pods | Manages ReplicaSets |
| No Rollback | Supports Rollback |
| No Rolling Update | Supports Rolling Updates |
| Rarely Used Directly | Most Common |

## Interview Questions

### What is a ReplicaSet?

ReplicaSet ensures a specified number of Pod replicas are always running.

---

### Why is ReplicaSet used?

For high availability, scaling, and self-healing.

---

### What happens if a Pod fails?

ReplicaSet automatically creates a replacement Pod.

---

### Difference between ReplicaSet and Deployment?

Deployment manages ReplicaSets and provides rolling updates and rollback functionality.

---

### Can ReplicaSet update application versions?

No.

Deployment is used for version updates.
