# ConfigMaps in Kubernetes

## What is a ConfigMap?

ConfigMap is a Kubernetes object used to store non-confidential configuration data in key-value pairs.

It helps separate application configuration from container images.

---

## Why Use ConfigMaps?

Without ConfigMap:

Application configuration is hardcoded inside the container.

Problems:
- Need to rebuild image for every change
- Difficult to manage environments
- Poor maintainability

With ConfigMap:

- Configuration stored separately
- No need to rebuild images
- Easy environment management
- Better portability

---

## Real Life Example

Imagine a mobile application.

Hardcoded Approach:

Backend URL:
https://prod-api.company.com

If URL changes:
- Modify code
- Build image again
- Deploy again

ConfigMap Approach:

Backend URL stored in ConfigMap.

Application reads configuration from ConfigMap.

Only ConfigMap needs update.

## ConfigMap Data Types

### 1. Literal Values

```yaml
data:
  APP_ENV: production
  APP_PORT: "3000"
```

### 2. Configuration Files

```yaml
data:
  app.properties: |
    db.host=mysql
    db.port=3306
```

---

## Creating ConfigMap

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: app-config
data:
  APP_ENV: production
  APP_PORT: "3000"
```

---

## How Pods Consume ConfigMaps?

Three ways:

1. Environment Variables
2. Command Line Arguments
3. Mounted Volumes

---

## Method 1: Environment Variables

```yaml
env:
- name: APP_ENV
  valueFrom:
    configMapKeyRef:
      name: app-config
      key: APP_ENV
```

---

## Method 2: Entire ConfigMap

```yaml
envFrom:
- configMapRef:
    name: app-config
```

---

## Method 3: Mount as Volume

```yaml
volumes:
- name: config-volume
  configMap:
    name: app-config
```

## Advantages

- Decouples configuration
- Easy updates
- Environment-specific settings
- Better DevOps practices
- Reusable configurations

---

## Limitations

- Not secure
- Stored in plain text
- Not suitable for passwords
- Use Secrets for sensitive data

---

## ConfigMap vs Secret

| Feature | ConfigMap | Secret |
|----------|----------|---------|
| Data Type | Non-sensitive | Sensitive |
| Encoding | Plain Text | Base64 |
| Passwords | No | Yes |
| API Keys | No | Yes |

## Interview Questions

### What is ConfigMap?

ConfigMap stores non-confidential application configuration in Kubernetes.

---

### Why use ConfigMap?

To separate configuration from application code.

---

### Difference between ConfigMap and Secret?

ConfigMap stores normal configuration.
Secret stores sensitive information.

---

### How many ways can Pods consume ConfigMaps?

- Environment Variables
- envFrom
- Mounted Volumes

---

### Can ConfigMaps store passwords?

No.
Use Secrets instead.

---

### What happens if ConfigMap changes?

Running Pods usually do not restart automatically.

Pod restart may be required depending on implementation.
