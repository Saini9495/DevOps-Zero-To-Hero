# Kubernetes Namespace

## What is Namespace?

A Namespace is a logical partition inside a Kubernetes cluster that allows multiple teams, applications, or environments to share the same cluster while remaining isolated from each other.

Namespaces help organize and manage resources efficiently.

---

## Why Do We Need Namespaces?

Imagine a company has:

- Development Team
- Testing Team
- Production Team

All teams use the same Kubernetes cluster.

Without Namespaces:

- Resource names may conflict
- Difficult to manage workloads
- Security issues
- Hard to monitor resources

With Namespaces:

- Resources remain isolated
- Better organization
- Easier management
- Improved security

---

## Default Namespaces in Kubernetes

### default

Used when no namespace is specified.

```bash
kubectl get pods
```

Pods are created in the default namespace.

---

### kube-system

Contains Kubernetes system components.

Examples:

- etcd
- kube-apiserver
- kube-scheduler
- CoreDNS

```bash
kubectl get pods -n kube-system
```

---

### kube-public

Accessible by all users.

Stores publicly readable cluster information.

---

### kube-node-lease

Used by nodes for heartbeat information.

Helps Kubernetes determine node availability.

## Creating a Namespace

```bash
kubectl create namespace dev
```

Verify:

```bash
kubectl get namespaces
```

---

## Creating Resources Inside a Namespace

```bash
kubectl run nginx \
--image=nginx \
-n dev
```

Verify:

```bash
kubectl get pods -n dev
```

---

## Viewing Resources in All Namespaces

```bash
kubectl get pods --all-namespaces
```

or

```bash
kubectl get pods -A
```

---

## Namespace Isolation

Namespaces isolate:

- Pods
- Services
- Deployments
- ConfigMaps
- Secrets

But do not isolate:

- Nodes
- Persistent Volumes
- Storage Classes

---

## Switching Default Namespace

Current Context:

```bash
kubectl config view --minify
```

Set Namespace:

```bash
kubectl config set-context --current --namespace=dev
```

Now every command will use the dev namespace.

---

## Deleting a Namespace

```bash
kubectl delete namespace dev
```

Warning:

Deleting a namespace deletes all resources inside it.
