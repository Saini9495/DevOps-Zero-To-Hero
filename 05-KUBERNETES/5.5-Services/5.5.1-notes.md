# Kubernetes Services

## What is a Service?

A Service is a Kubernetes object that provides a stable network endpoint for Pods.

Pods are ephemeral in nature.

When a Pod gets deleted and recreated, its IP changes.

Service solves this problem by providing a fixed IP and DNS name.

---

## Why Do We Need Services?

Without Service:

Client → Pod IP

Problem:
- Pod IP changes after restart.
- Application becomes inaccessible.

With Service:

Client → Service → Pod

Benefits:
- Stable IP
- Load Balancing
- Service Discovery
- External Access

---

## How Service Works?

Service uses Labels and Selectors.

Example:

Pods Label

app=nginx

Service Selector

app=nginx

The Service automatically forwards traffic to all matching Pods.

## Types of Kubernetes Services

1. ClusterIP
2. NodePort
3. LoadBalancer

---

# ClusterIP

Default Service Type.

Accessible only inside the Kubernetes Cluster.

Use Cases:

- Backend APIs
- Databases
- Internal Communication

Architecture

Frontend Pod
      |
ClusterIP Service
      |
Backend Pods

Advantages

- Internal communication
- Secure
- Default service type

Disadvantages

- Not accessible from outside cluster

---

# NodePort

Exposes application on a port of every worker node.

Port Range:

30000 - 32767

Architecture

Browser
   |
NodeIP:30007
   |
NodePort Service
   |
Pods

Advantages

- Accessible from outside cluster
- Easy testing

Disadvantages

- Limited port range
- Not preferred for production

---

# LoadBalancer

Used mainly in cloud providers.

AWS
Azure
GCP

Architecture

Internet
    |
Load Balancer
    |
Service
    |
Pods

Advantages

- Production ready
- External IP
- Automatic Load Balancing

Disadvantages

- Cloud cost involved

---

## Service Discovery

Kubernetes automatically creates DNS.

Example

nginx-service.default.svc.cluster.local

Pods can communicate using Service Name.

Example

curl http://nginx-service

---

## Endpoints

Service doesn't directly connect to Pods.

It creates Endpoints.

Check:

kubectl get endpoints

---

## Real Life Example

Restaurant Example

Pods = Waiters

Service = Reception Counter

Customers don't search individual waiters.

Customers contact Reception.

Reception routes requests to available waiters.

## Interview Questions

### What is a Service?

A Service provides a stable network endpoint for Pods.

---

### Why do we need Services?

Because Pod IPs change frequently.

---

### Difference Between ClusterIP and NodePort?

ClusterIP:
Internal Access

NodePort:
External Access

---

### Difference Between NodePort and LoadBalancer?

NodePort:
Uses Node IP and Port

LoadBalancer:
Uses Cloud Load Balancer

---

### What are Endpoints?

Endpoints are actual Pod IPs managed by Service.

---

### What is Service Discovery?

Finding applications using DNS names instead of IP addresses.
