# Kubernetes Deployments

## What is a Deployment?

A Deployment is a Kubernetes object that manages Pods through ReplicaSets.

It provides:

- Scaling
- Self Healing
- Rolling Updates
- Rollbacks

Deployment is the most commonly used Kubernetes object for deploying applications.

---

## Why Do We Need Deployments?

Suppose you create a Pod directly.

Problem:

- If Pod crashes → Kubernetes recreates it.
- But if you need multiple Pods → Manual work.
- Updating image versions becomes difficult.

Deployment solves these problems.

---

## Architecture

User
 ↓
Deployment
 ↓
ReplicaSet
 ↓
Pods

Deployment manages ReplicaSets.

ReplicaSets manage Pods.

---

## How Deployment Works?

Step 1:
Create Deployment

Step 2:
Deployment creates ReplicaSet

Step 3:
ReplicaSet creates Pods

Step 4:
If Pod dies,
ReplicaSet creates a new Pod automatically

---

## Features

### 1. Scaling

Increase or decrease number of Pods.

Example:

3 Pods → 10 Pods

without downtime.

---

### 2. Rolling Updates

Update application version gradually.

Old Pods are replaced one by one.

No downtime.

---

### 3. Rollback

If new version fails,

Deployment can revert to previous version.

---

### 4. Self Healing

If Pod crashes,

ReplicaSet automatically recreates it.

---

## Deployment Lifecycle

Create Deployment
        ↓
Create ReplicaSet
        ↓
Create Pods
        ↓
Monitor Pods
        ↓
Scale / Update / Rollback

---

## Real Life Example

Imagine a restaurant.

Deployment = Restaurant Manager

ReplicaSet = Supervisor

Pods = Workers

Manager decides:

"We need 5 workers."

Supervisor ensures:

"Exactly 5 workers are always available."

If a worker leaves,

Supervisor hires another worker automatically.

---

## Deployment Strategy

### Recreate

Old Pods deleted first.

New Pods created later.

Downtime occurs.

---

### Rolling Update

Default strategy.

Old Pods replaced gradually.

No downtime.

---

## Deployment YAML Components

### apiVersion

Defines API version.

```yaml
apiVersion: apps/v1
```

### kind

Defines resource type.

```yaml
kind: Deployment
```

### metadata

Deployment information.

```yaml
metadata:
  name: nginx-deployment
```

### spec

Desired state.

```yaml
spec:
```

### replicas

Number of Pods.

```yaml
replicas: 3
```

### selector

Selects Pods.

```yaml
selector:
  matchLabels:
    app: nginx
```

Blueprint for Pods.
