# Helm in Kubernetes

## What is Helm?

Helm is a package manager for Kubernetes.

Just like:

- apt for Ubuntu
- yum for CentOS
- npm for Node.js

Helm is used to install, upgrade, rollback and manage Kubernetes applications using pre-defined templates called Charts.

---

## Why Helm?

Without Helm:

Deployment.yml
Service.yml
ConfigMap.yml
Secret.yml
Ingress.yml

Har file manually manage karni padti hai.

With Helm:

Single command se pura application deploy ho jata hai.

```bash
helm install myapp ./my-chart
```

## Important Terminologies

### Chart

Helm package containing Kubernetes resources.

Example:

Deployment
Service
Ingress
ConfigMap

sab ek Chart ke andar hote hain.

---

### Repository

Charts store karne ki location.

Examples:

Bitnami Repository

Artifact Hub

Custom Repository

---

### Release

Chart ka deployed instance.

Example:

Chart Name = nginx

Release Name = my-nginx

```bash
helm install my-nginx bitnami/nginx
```

my-nginx is Release.

---

### Values.yaml

Configuration file.

Example:

```yaml
replicaCount: 3

image:
  repository: nginx
  tag: latest
```

## Helm Chart Structure

my-chart/

```text
my-chart/
│
├── Chart.yaml
├── values.yaml
├── charts/
├── templates/
│   ├── deployment.yaml
│   ├── service.yaml
│   ├── ingress.yaml
│   └── configmap.yaml
│
└── .helmignore
```

---

## Chart.yaml

Contains chart metadata.

Example:

```yaml
apiVersion: v2
name: nginx-chart
description: Helm Chart for Nginx
version: 0.1.0
appVersion: "1.0"
```

---

## values.yaml

Contains configurable values.

Example:

```yaml
replicaCount: 2

image:
  repository: nginx
  tag: latest

service:
  type: ClusterIP
  port: 80
```

---

## Templates Folder

Contains Kubernetes YAML templates.

Example:

deployment.yaml

service.yaml

ingress.yaml

## Interview Questions

### What is Helm?

Helm is a package manager for Kubernetes used to deploy and manage applications using Charts.

---

### What is a Chart?

A collection of Kubernetes manifests packaged together.

---

### What is a Release?

A running instance of a Chart inside a Kubernetes cluster.

---

### What is values.yaml?

Configuration file used to customize Helm Charts.

---

### Difference Between Chart and Release?

Chart = Package

Release = Installed Package

---

### How Rollback Works in Helm?

Using revision history.

```bash
helm rollback RELEASE_NAME REVISION
```

---

### Why Helm is Used?

To simplify Kubernetes application deployment and management.
